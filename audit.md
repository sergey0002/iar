### 🔴 КРИТИЧНО — SEV-1

#### [SEC-01] Path Traversal — Частичная защита с риском обхода

**Файл:** `index.php`, строки 22–28  
**Текущий код:**

```php
$filePath = realpath($baseDir . DIRECTORY_SEPARATOR . $name);
if (!$filePath || !str_starts_with($filePath, $baseDir)) {
```

**Уязвимость:**  
`$baseDir` получается из `$config['images_base_dir']` = `realpath(__DIR__ . '/../')`. Значение **не нормализовано через `realpath()`** в `index.php` — оно используется сырым. Если возвращаемое значение из `config.php` не обработано через `realpath()`, то на Windows с нестандартным регистром букв в пути `str_starts_with()` может дать ложное `true` (`C:\Laragon\www` vs `c:\laragon\www`).

**Дополнительно:** Функция `str_starts_with($filePath, $baseDir)` без завершающего `/` на конце `$baseDir` уязвима к атаке вида: если `$baseDir = /var/www/site`, то путь `/var/www/site-evil/file.jpg` тоже пройдёт проверку!

**Исправление:**

```php
$baseDir = rtrim(realpath($config['images_base_dir']), DIRECTORY_SEPARATOR) . DIRECTORY_SEPARATOR;
$filePath = realpath($baseDir . $name);
if (!$filePath || strncmp($filePath, $baseDir, strlen($baseDir)) !== 0) {
    header("HTTP/1.1 403 Forbidden");
    die("Access denied.");
}
```

#### [SEC-02] Гравитация и режим не валидируются

**Файл:** `index.php`, строки 43–44; `ResizeService.php`, строки 102–103  
**Текущий код:**

```php
$mode    = $_GET['m'] ?? $config['default_mode'];
$gravity = $_GET['g'] ?? $config['default_gravity'];
```

**Уязвимость:**  
Параметры `m` и `g` передаются напрямую в `Intervention Image` без валидации. Невалидные или специально сформированные значения (например `m=../../../../etc` или строки с нулевыми байтами) могут вызвать непредсказуемое поведение библиотеки или исключение, которое будет экранировано, но всё равно создаёт риск информационной утечки в `error_log`.

**Исправление:**

```php
$allowedModes    = ['fit', 'cover', 'resize'];
$allowedGravity  = ['center', 'top', 'bottom', 'left', 'right', 'top-left', 'top-right', 'bottom-left', 'bottom-right'];
$mode    = in_array($_GET['m'] ?? '', $allowedModes, true) ? $_GET['m'] : $config['default_mode'];
$gravity = in_array($_GET['g'] ?? '', $allowedGravity, true) ? $_GET['g'] : $config['default_gravity'];
```

### 🟠 ВАЖНО — SEV-2

#### [SEC-03] ETag вычисляется через md5_file() при каждом запросе к кешу

**Файл:** `index.php`, строка 145  
**Текущий код:**

```php
header("ETag: " . md5_file($path));
```

**Проблема:**  
`md5_file()` читает весь файл для вычисления хеша, хотя файл уже лежит на диске. Для больших изображений (2000×2000 px) это добавляет лишние I/O-операции при каждом cache-hit запросе. Не является уязвимостью, но при высокой нагрузке создаёт CPU/IO bottleneck.

**Исправление:**  
Использовать `filemtime()` + `filesize()` как ETag:

```php
header('ETag: "' . md5(filemtime($path) . filesize($path)) . '"');
```

#### [SEC-04] Параметр `fm` (format) не валидируется

**Файл:** `index.php`, строка 62  

```php
$outputFormat = $_GET['fm'] ?? null;
```

**Проблема:**  
Значение используется для построения имени файла кеша без проверки. Хотя `$cacheFile` не исполняется, нежелательные значения (`fm=../../etc/passwd`) могут создать файлы с неожиданными расширениями в `cache/`.

**Исправление:**

```php
$outputFormat = null;
if (isset($_GET['fm'])) {
    $fm = strtolower(preg_replace('/[^a-z]/', '', $_GET['fm']));
    if (in_array($fm, $config['allowed_extensions'], true)) {
        $outputFormat = $fm;
    }
}
```

#### [SEC-05] crop-параметры не ограничены сверху в index.php

**Файл:** `index.php`, строки 48–58  

**Проблема:**  
Значение кроп-параметра (`ct`, `cr`, `cb`, `cl`) принимается как `float`, приводится к числу и передаётся в `applyEdgeCrop()`. В `ResizeService` есть защита (90% максимум), но значение `?ct=999999999p` дойдёт до сервиса и будет обработано (без вреда, но с ненужными вычислениями).

**Исправление (index.php):**

```php
$value = min((float)$val, 10000); // Ограничение сверху
```

#### [SEC-06] Отсутствует Rate Limiting / защита от DoS на уровне приложения

**Файл:** `index.php`  

**Проблема:**  
Нет ограничения на количество запросов с одного IP. Злоумышленник может сгенерировать тысячи уникальных запросов с разными `w`, `h`, `m`, `g`, создавая огромное количество файлов в `cache/` и исчерпывая дисковое пространство.

- Или добавить простой счётчик через `APCu` / файловую семафорную блокировку
- Ограничить набор допустимых размеров (`allowed_sizes` whitelist в config)

### 🟡 ЗАМЕЧАНИЕ — SEV-3

#### [SEC-07] SVG не санируется перед копированием и кешированием

**Файл:** `ResizeService.php`, строка 179  

```php
return copy($sourcePath, $cachePath);
```

**Проблема:**  
SVG-файл копируется в кеш **без санации**. SVG — это XML, который может содержать `<script>`, `<iframe>`, `xlink:href` и другие векторы XSS. Если `cache/` доступен через HTTP напрямую (без прохождения через `index.php`), браузер рендерит SVG со скриптами.

**Анализ:**  
Текущий `.htaccess` перехватывает только запросы к несуществующим файлам. Если `cache/` доступен напрямую — это риск.

**Рекомендации:**

1. Добавить в `cache/.htaccess`: `Header set Content-Security-Policy "default-src 'none'"`
2. Или использовать библиотеку `svg-sanitizer` перед копированием
3. Хранить svg-кеш с заголовком `Content-Disposition: attachment` для старых браузеров

**Файл:** `config.php`, строка 24

**Рекомендация:**  
Явно ограничить `images_base_dir` конкретным path-ом, а не полагаться на `realpath(__DIR__ . '/../')`.

### 🔴 Критичные

#### [IMP-01] `resize_class.php` — мёртвый код, создаёт угрозу

**Файл:** `resize_class.php`

**Проблема:**  
Устаревший класс `resize` (PHP5, GD raw API, без namespace) **не используется нигде** в текущей системе, но присутствует в корне. Создаёт путаницу и потенциально может быть вызван при добавлении нового функционала.

**Уязвимости внутри класса:**

- `@imagecreatefrompng()` — подавление ошибок через `@`
- Нет проверки допустимых расширений (только jpg/gif/png — без webp/avif)
- Нет защиты от Path Traversal
- `imagedestroy()` — устаревшее с PHP 8.0 (deprecated)

**Рекомендация:** Удалить `resize_class.php` из проекта.

#### [IMP-02] Обрезка поддерживает только одну сторону — план декларирует это, но не обосновывает

**Файл:** `index.php`, строки 46–58; `ResizeService.php`, строка 54

**Проблема:**  
Допускается только один параметр кропа за запрос. Комбинированная обрезка (`ct=50&cb=30` — срезать сверху и снизу) **невозможна**. Это существенное функциональное ограничение при удалении водяных знаков или оформлении изображений.

**Рекомендация:**  
Расширить до массива кроп-операций или поддержать одновременно все четыре стороны.

#### [IMP-03] `composer.json` не включает `ext-gd` как зависимость

**Файл:** `composer.json`, строки 5–8  

```json

    "php": ">=7.4",

```

**Проблема:**  
GD — критическая зависимость. Её отсутствие не вызовет ошибки при `composer install`, а только при первом запросе изображения (runtime exception). Это затрудняет диагностику.

#### [IMP-04] Нет ограничения режима 'fit' при нулевых размерах

#### [IMP-05] SVG с fallback-копированием не учитывает `allowed_extensions` для выходного формата

**Файл:** `ResizeService.php`, строки 177–179  

```php
copy($sourcePath, $cachePath);
return true;
```

**Проблема:**  
Когда Imagick недоступен, SVG копируется в кеш **независимо** от запрошенного `$targetExt`. Если клиент запросил `?fm=jpg`, будет кешировано и отдано SVG с заголовком `Content-Type: image/jpeg` — это неверно и сломает браузер.

**Рекомендация:**

```php
if ($targetExt !== 'svg') {
    // Нет Imagick — нельзя растеризовать, вернуть ошибку
    error_log('SVG rasterization requested but Imagick not available');
    return false;
}
copy($sourcePath, $cachePath);
return true;
```

#### [IMP-07] `cache_auto_clean` использует `glob()` — может быть медленным

**Файл:** `index.php`, строки 103–108  

```php
$pattern = $config['cache_dir'] . DIRECTORY_SEPARATOR . pathinfo($filePath, PATHINFO_FILENAME) . '__' . $w . 'x' . $h . '*' . $pathHash . '_*';
foreach (glob($pattern) as $oldCache) {
    @unlink($oldCache);
}
```

**Проблема:**  
При тысячах файлов в `cache/` операция `glob()` будет сканировать всю директорию на каждый мутирующий запрос. На HDD-серверах это заметная задержка.

**Рекомендация:**  
Выполнять автоочистку асинхронно (через отдельный cronjob) или ограничить количество файлов в `cache/` через иерархическую структуру директорий.

#### [IMP-12] Нет HTTP `Vary: Accept` заголовка при auto_format

**Файл:** `index.php`, `serveFile()`  

**Проблема:**  
При использовании `auto_format` разные клиенты могут получать разные форматы по одному URL. Промежуточные кеши (CDN, proxy) должны быть уведомлены об этом через заголовок:

```
Vary: Accept
```

Без него CDN закеширует WebP-версию и отдаст её Safari-клиентам, которые WebP не поддерживают.

**Исправление:**

```php
if ($config['auto_format'] ?? true) {
    header('Vary: Accept');
}
```

#### [IMP-13] `download_composer.php` — потенциально опасный скрипт в корне

**Файл:** `download_composer.php`  

**Проблема:**  
Если этот файл доступен публично через HTTP, он позволяет скачать/перезаписать `composer.phar` с удалённого источника. Это вектор атаки: если злоумышленник может повлиять на DNS или CDN Composer, он может заменить `composer.phar`.

**Рекомендация:**  
Удалить файл после первоначальной установки. Добавить в `.htaccess`:

```apache
<Files "download_composer.php">
    Order deny,allow
    Deny from all
</Files>
```

## Раздел 4: Аудит `.htaccess`

```apache
RewriteBase /thubms/
RewriteRule ^(.*)\.(jpg|...svg)$ index.php?name=$1.$2 [L,QSA]
```

| Проблема | Описание |
|----------|----------|
| `RewriteBase /thubms/` | Хардкод пути! При деплое в корень или другой путь — сломается |
| Нет `RewriteCond %{REQUEST_FILENAME} !-f` для `cache/` | Кешированные файлы также идут через `index.php` вместо прямой отдачи |
| Нет защиты от прямого доступа к `cache/` SVG-файлам | XSS-риск (см. SEC-07) |
| `QSA` флаг без экранирования | Параметры `name` могут быть повторно интерпретированы |

**Рекомендуемые дополнения:**

```apache
# Прямая отдача кешированных файлов (без PHP):
RewriteCond %{REQUEST_URI} ^/thubms/cache/
RewriteCond %{REQUEST_FILENAME} -f
RewriteRule ^ - [L]

# Защита cache/ от прямых SVG-запросов:
<FilesMatch "\.svg$">
    Header set Content-Security-Policy "default-src 'none'"
</FilesMatch>
```
