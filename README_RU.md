## 🌟 Почему именно iar?

| | Преимущество |
|:---:|:---|
| ⚡ | **Быстрая отдача** — Прозрачное кеширование + ETag + опциональный редирект 302/301 на статичный файл кеша. Повторные запросы — как статика. |
| 🛡️ | **Безопасен по умолчанию** — Защита от Path Traversal и DoS. Белые списки расширений и строгая проверка путей. |
| 📦 | **Ноль системных зависимостей** — Нужен только PHP + GD. Не требует `imagemagick` или глобального `composer` на сервере. |
| 🚀 | **Современные форматы** — Автоматически отдаёт WebP или AVIF, если браузер поддерживает (`Accept` header). |
| 📁 | **Полная поддержка подпапок** — Работает с любой глубиной вложенности директорий. |
| 🔧 | **Единая точка конфига** — Все настройки в одном файле `config.php`. Менять код не нужно. |

## 🚀 Быстрый старт

### Установка

`iar` требует **Intervention Image v3**. Вы ДОЛЖНЫ установить зависимости через Composer **внутри** папки проекта.

**Способ А — через Git (рекомендуется):**

```bash
git clone https://github.com/sergey0002/iar.git
cd iar
composer install --no-dev --optimize-autoloader
```

**Способ Б — вручную:**

1. Скачайте и скопируйте папку `iar` в корень вашего сайта.
2. Откройте терминал и перейдите **внутрь** папки `iar`:

   ```bash
   cd /путь/к/сайту/iar
   ```

3. Установите зависимости.

   *Если глобальный `composer` установлен:*

   ```bash
   composer install --no-dev --optimize-autoloader
   ```

   *Если `composer` не установлен глобально — скачайте его локально:*

   ```bash
   php -r "copy('https://getcomposer.org/installer', 'cs.php');"
   php cs.php --filename=composer.phar --quiet
   php -r "unlink('cs.php');"
   php composer.phar install --no-dev --optimize-autoloader
   ```

> [!NOTE]
> Папка `cache/` создаётся **автоматически** при первом запросе. Убедитесь, что PHP-процесс имеет права на запись в директорию `iar/`.

> [!TIP]
> Папку `iar/` можно **переименовать** на любое другое имя — всё будет работать корректно. Правила `.htaccess` относительные и переносимые.

## 📖 Примеры использования

> **Принцип:** обращайтесь к изображению по его оригинальному пути через папку `iar/`. Система сама найдёт файл, обработает и закеширует результат.

### Базовый ресайз

`/products/hero.jpg` — реальный путь к изображению в корне сайта.

```html
<!-- Ресайз до 800px по ширине, высота пропорционально -->
<img src="/iar/products/hero.jpg?w=800">

<!-- Только по высоте -->
<img src="/iar/products/hero.jpg?h=600">
```

### Адаптивные изображения (`srcset`)

```html
<img
  src="/iar/products/hero.jpg?w=1200"
  srcset="
    /iar/products/hero.jpg?w=480 480w,
    /iar/products/hero.jpg?w=800 800w,
    /iar/products/hero.jpg?w=1200 1200w
  "
  sizes="(max-width: 600px) 480px, (max-width: 1000px) 800px, 1200px"
  alt="SEO-оптимизированное фото"
  loading="lazy">
```

### Изображения в подпапках

```html
<!-- /catalog/phones/2026/iphone15.jpg — оригинальный путь -->
<img src="/iar/catalog/phones/2026/iphone15.jpg?w=400&h=400&m=cover">

<!-- Глубокая вложенность -->
<img src="/iar/assets/img/ui/icons/logo.png?w=100">
```

### Квадратный аватар с умным кропом

```html
<!-- Кроп по центру (дефолт), 200×200 -->
<img src="/iar/users/avatar.jpg?w=200&h=200&m=cover">

<!-- Кроп с акцентом на верхней части (портрет) -->
<img src="/iar/users/profile.jpg?w=300&h=300&m=cover&g=top">
```

### Обложка статьи с обрезкой краёв

```html
<!-- Срезать 15% снизу (убрать водяной знак), потом ресайз -->
<img src="/iar/blog/cover.jpg?w=1200&h=400&cb=15p&m=cover&g=top">

<!-- Срезать 50px сверху и ресайз -->
<img src="/iar/photo.jpg?w=640&h=480&ct=50">
```

### Принудительная конвертация формата

```html
<!-- Всегда отдавать WebP, независимо от браузера -->
<img src="/iar/logo.png?w=200&fm=webp">

<!-- Конвертировать в AVIF -->
<img src="/iar/photo.jpg?w=800&fm=avif">
```

## 🔗 URL API — Полный справочник параметров

### Основные параметры

| Параметр | Тип | По умолчанию | Описание |
| :--- | :---: | :---: | :--- |
| *(путь в URL)* | string | — | Путь к оригиналу, передаётся как суффикс URL: `/iar/путь/к/файлу.jpg` |
| `w` | int | `0` | Ширина в px. `0` = не ограничивать |
| `h` | int | `0` | Высота в px. `0` = не ограничивать |

### Режим масштабирования (`m`)

| Значение | Поведение |
| :--- | :--- |
| `fit` *(по умолчанию)* | Пропорционально вписывает в `w×h`. Не обрезает. Не увеличивает маленькие изображения. |
| `cover` | Заполняет всю область `w×h` без искажений. Края обрезаются по точке `g`. |
| `resize` | Строго растягивает в `w×h`, игнорируя пропорции. Может искажать. |

### Гравитация (`g`) — только для `m=cover`

| Значение | Точка привязки |
| :--- | :--- |
| `center` *(по умолчанию)* | Центр |
| `top` | Верх по центру |
| `bottom` | Низ по центру |
| `left` | Левый центр |
| `right` | Правый центр |
| `top-left` | Верхний левый угол |
| `top-right` | Верхний правый угол |
| `bottom-left` | Нижний левый угол |
| `bottom-right` | Нижний правый угол |

### Обрезка краёв (`ct`, `cr`, `cb`, `cl`)

Обрезает указанное количество пикселей или процентов с края **перед** ресайзом.

| Параметр | Край | В пикселях | В процентах |
| :---: | :--- | :---: | :---: |
| `ct` | Верх (top) | `?ct=50` | `?ct=15p` |
| `cr` | Право (right) | `?cr=10` | `?cr=10p` |
| `cb` | Низ (bottom) | `?cb=30` | `?cb=20p` |
| `cl` | Лево (left) | `?cl=20` | `?cl=5p` |

> [!NOTE]
> Суффикс `p` означает проценты. Одновременно активен только **один** параметр: приоритет `ct → cr → cb → cl`.

### Формат вывода (`fm`)

| Значение | Описание |
| :--- | :--- |
| `webp` | Принудительно конвертировать в WebP |
| `avif` | Принудительно конвертировать в AVIF |
| `jpg` | Сохранить как JPEG |
| `png` | Сохранить как PNG |

> Если `fm` не передан и в конфиге включён `auto_format`, скрипт автоматически выберет лучший формат по заголовку `Accept` браузера.

## ⚙️ Конфигурация `config.php`

Единственная точка настройки. Менять код в других файлах не нужно.

```php
return [
    // === Качество и размеры ===
    'max_width'       => 2000,   // Максимальная ширина (защита от DoS)
    'max_height'      => 2000,   // Максимальная высота
    'jpeg_quality'    => 82,     // JPEG: 1–100 (82 = стандарт индустрии)
    'webp_quality'    => 85,     // WebP: 1–100
    'avif_quality'    => 75,     // AVIF: 1–100
    'png_compression' => 9,      // PNG zlib: 0–9 (не влияет на качество пикселей)

    // === Оптимизация PNG ===
    'png_indexed'           => true,  // Умный indexed PNG для маленьких (экономит 30–70%)
    'png_indexed_threshold' => 200,   // px: шире этого — остаётся true-color; 0 = всегда

    // === Оптимизация JPEG ===
    'jpeg_progressive' => true,       // Прогрессивный JPEG (плавная загрузка сверху вниз)

    // === Оптимизация WebP ===
    'webp_strip_metadata' => true,    // Удалять EXIF/XMP из WebP (экономит 1–5 KB)

    // === Режимы по умолчанию ===
    'default_mode'    => 'fit',       // fit | cover | resize
    'default_gravity' => 'center',    // center | top | bottom | left | right | ...

    // === Оптимизация и кеширование ===
    'lossless'           => false,    // true = максимальное качество (quality 100)
    'auto_format'        => true,     // Авто-выбор WebP/AVIF по Accept-заголовку
    'cache_invalidation' => 'mtime',  // 'mtime' (по дате оригинала) | 'off' (вечный кеш)
    'cache_auto_clean'   => true,     // Удалять старые версии кеша при обновлении
    'redirect_to_cache'  => true,     // true = редирект 302/301 на статичный файл кеша
    'static_mode'        => false,    // true = без проверки mtime, 301 редирект (макс. скорость)
    'disable_cache'      => false,    // true = отключить кеш (перегенерация всегда, для разработки)

    // === Безопасность и пути ===
    'images_base_dir'    => realpath(__DIR__ . '/../'),
    'allowed_extensions' => ['jpg', 'jpeg', 'gif', 'png', 'webp', 'avif', 'svg'],
    'cache_dir'          => __DIR__ . '/cache',

    // === Логирование ===
    'logging' => true,
    'log_dir' => __DIR__ . '/log',
];
```

### Описание всех параметров конфигурации

| Ключ | Значение | Описание |
| :--- | :---: | :--- |
| `max_width` / `max_height` | int, px | Верхний предел размеров. Значения `w`/`h` больше лимита обрезаются. **Защита от DoS.** |
| `jpeg_quality` | 1–100 | Качество JPEG. `82` — визуально без потерь, оптимальный размер файла. |
| `webp_quality` | 1–100 | Качество сжатия WebP (lossy). `85` — рекомендуется. |
| `avif_quality` | 1–100 | Качество AVIF. `75` — отличный баланс качества и размера. |
| `png_compression` | 0–9 | Уровень zlib-сжатия PNG. **Не влияет на качество пикселей** (always lossless). `9` = минимальный размер. |
| `png_indexed` | bool | Умный режим индексированных цветов (256-цветная палитра) для PNG. Экономит 30–70% для иконок и flat-арта. |
| `png_indexed_threshold` | px | Ширина, выше которой indexed-режим отключается (изображение остаётся true-color). `0` = всегда indexed. |
| `jpeg_progressive` | bool | Прогрессивный JPEG — картинка загружается плавно сверху вниз в браузере. |
| `webp_strip_metadata` | bool | Удалять EXIF/XMP метаданные из WebP. Экономит 1–5 KB на файл. |
| `default_mode` | `fit` / `cover` / `resize` | Режим масштабирования по умолчанию, если `m` не передан в URL. |
| `default_gravity` | `center` / ... | Точка кропа по умолчанию для режима `cover`, если `g` не передан. |
| `lossless` | bool | `true` = все форматы в максимальном качестве (JPEG/WebP/AVIF quality=100, PNG без сжатия). |
| `auto_format` | bool | Автоматически отдавать WebP / AVIF, если браузер сообщает о поддержке через `Accept`. |
| `cache_invalidation` | `mtime` / `off` | `mtime` — пересоздавать кеш при изменении оригинала. `off` — кеш вечный. |
| `cache_auto_clean` | bool | Удалять старые версии кеш-файла при обновлении оригинала. |
| `redirect_to_cache` | bool | `true` = PHP делает 302/301 редирект на статичный файл в `cache/` (Apache отдаёт напрямую). `false` = поток через PHP. |
| `static_mode` | bool | `true` = пропустить проверку `filemtime` полностью (минимум I/O), 301 редирект. Для неизменяемых ресурсов. |
| `disable_cache` | bool | `true` = отключить кеш (всегда перегенерировать). Посылает заголовки `no-cache`. **Только для разработки.** |
| `images_base_dir` | path | Корневая директория оригиналов. Запросы за её пределы — 403. |
| `allowed_extensions` | array | Белый список расширений. Всё остальное возвращает 403. |
| `cache_dir` | path | Папка для хранения кешированных миниатюр. Создаётся автоматически. |
| `logging` | bool | Включить запись логов в папку `log/`. |
| `log_dir` | path | Директория для лог-файлов (один файл в день). |

### Сравнение режимов кеша

| Режим | Конфиг | Поведение | Когда использовать |
| :--- | :--- | :--- | :--- |
| **Обычный** | по умолчанию | Проверка mtime + 302 редирект | Изменяемые ресурсы |
| **Статичный** | `static_mode: true` | Без проверки mtime + 301 редирект | Неизменяемые ресурсы |
| **Разработка** | `disable_cache: true` | Всегда перегенерировать | Разработка |

## ✨ Возможности системы

| Возможность | Описание |
| :--- | :--- |
| **Ресайз** | Пропорциональный ресайз по ширине (`w`) или высоте (`h`). Не увеличивает маленькие изображения. |
| **Умный кроп** | Режим `cover` с выбором точки фокуса (`g`). |
| **Обрезка краёв** | Срезать px или % с любого края перед обработкой (`ct`, `cr`, `cb`, `cl`). |
| **Авто WebP/AVIF** | Прозрачная конвертация в современные форматы без изменения URL. |
| **Прозрачное кеширование** | Первый запрос = обработка. Все последующие = мгновенная отдача из кеша. |
| **Редирект на кеш** | PHP делает редирект на статичный файл кеша — Apache отдаёт напрямую без PHP. |
| **Статичный режим** | Нулевые I/O-накладные расходы для неизменяемых ресурсов. |
| **Режим разработки** | `disable_cache` — всегда свежие изображения без ручной очистки кеша. |
| **Инвалидация кеша** | Автоматически при изменении оригинала (по `mtime`). |
| **ETag / 304** | Полная поддержка браузерного кеширования. |
| **Прогрессивный JPEG** | Плавная загрузка сверху вниз. |
| **Умный PNG indexed** | Автоматический режим палитры для маленьких PNG (экономия 30–70%). |
| **Стриппинг WebP** | Удаление EXIF/XMP из WebP на лету. |
| **Поддержка SVG** | SVG передаётся как есть (или растеризуется через Imagick). |
| **Path Traversal защита** | Строгая проверка пути — невозможно выйти за пределы `images_base_dir`. |
| **DoS защита** | Лимиты `max_width` / `max_height` предотвращают генерацию гигантских изображений. |
| **Lossless режим** | Глобальное максимальное качество одним параметром конфига. |
| **Логирование** | Дневные лог-файлы: статус кеша, тип редиректа, экономия размера. |
